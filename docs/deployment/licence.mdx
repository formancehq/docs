## To install the license using [Helm Global](https://helm.sh/docs/chart_template_guide/subcharts_and_globals/#global-chart-values) 

> All [Formance charts](https://github.com/formancehq/helm/) follow the same convention as described below.

Add your licence token and clusterID to the `values.yaml` file.

```yaml
global:
  licence:
    token: "your-token-here"
    clusterID: "your-cluster-id-here"
...
```

### Licence configuration

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| global.licence.clusterID | string | `""` | Obtain your licence cluster id with `kubectl get ns kube-system -o jsonpath='{.metadata.uid}' (Optional in the operator)` |
| global.licence.existingSecret | string | `""` | Licence Client Token as a secret |
| global.licence.issuer | string | `"https://license.formance.cloud/keys"` | Licence Environment  |
| global.licence.secretKeys.token | string | `""` | Key in existing secret to use for Licence Client Token |
| global.licence.token | string | `""` | Licence Client Token delivered by contacting [Formance](https://formance.com) |

## Operator additional requirement

As using Operator Resource Reference in order to copy the secret, add the label `formance.com/stack: any` to the licence secret in order for the Operator to copy the secret in the stack namespace.

### Debug

Once installed you must see the following logs in any EE pod:

```bash
│ {"level":"info","msg":"Licence check started","time":"2025-02-20T10:17:22Z"}                                                                                                            │
│ {"level":"info","msg":"Licence check passed","time":"2025-02-20T10:17:22Z"}      
```

Any errors will be logged as well.

```bash
│ {"level":"error","msg":"Licence check failed: invalid token","time":"2025-02-20T10:25:22Z"}     
```
